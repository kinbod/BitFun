# 洞察汇报（Insight Report）

## 风格标识
`insight-report`

## 适用场景
学术汇报、行业研究、咨询分析、内部洞察/复盘汇报、评测与尽调报告、技术方案深拆等需要详尽论证与高信息密度的工作/研究场景演示。

## 风格定位
这是「分析文档上墙」，不是「演讲幻灯」。读者是会前后会逐行细读的领导、评审和同行，每一页都必须经得起脱离演讲者的独立阅读。每页聚焦一个主题点，用完整句子、固定分析框架、满版数据表和过程叙事把这个点**讲透**。生成时优先级排序：信息量与论证完整 > 逻辑显性 > 版面美观。**在不溢出画布的前提下，内容越多越密越好；留白在本风格中是浪费，不是美德。**

## 画布填充硬指标（最高优先级，逐页强制自检）
- **垂直填充率 ≥ 90%**：正文内容必须从标题分隔线下方一直排到页底脚注线上方。若内容结束位置距脚注线还有超过 40pt 的空白，该页不合格——必须继续增加模块、行、列或展开分析，直到填满。
- **内容页正文 450–900 个汉字**（或等量英文）为目标区间；低于 300 字的内容页不合格。封面页也不许空：用 120–250 字摘要段 + 指标条 + 范围/方法/来源说明填满，禁止「大标题 + 半页空白」。
- 一个主题的素材不足以填满一页时，**不许留白**，按以下顺序补密度：(1) 增加分析维度（风险、反例、边界条件、竞品对照）；(2) 给每条结论补具体证据（数字、日期、文件、原话）；(3) 增加「So what / 本页结论」横条；(4) 与相邻主题合并成一页。
- 每页底部、脚注线之上，放一条**「本页结论」横条**（浅灰底或左侧主色粗边线，1–2 个完整句子），既补密度又强制每页有判断。封面与章节页除外。

## 信息密度与语言
- 用**完整句子**写分析，禁止电报式名词短语。每条要点都是「主语 + 判断 + 限定/证据」、可独立阅读的句子。例如写「它只看到了"插桩"这一个动作，但没有预见到这个动作在 MUSL 环境下、在初始化前执行的时序问题」，而不是只写「时序问题」。
- **每个卡片/单元格/小节都必须是多行实体**：卡片至少 4–7 行（一个加粗小标题 + 3 组以上「标签：完整句子」字段）；只有 1–2 行的卡片不合格，要么补字段要么合并。
- 鼓励 2–3 级嵌套列表：第一级是阶段或分论点，第二级是固定分析维度，第三级是具体证据或例子。层级靠缩进和列表符号区分。
- 行内强调系统承担「快速扫读」功能，这是高密度可读的关键：
  - 关键概念、机制、判断词**加粗**（`font-weight: 700`）；
  - 失败、风险、转折点、用户介入等警示信息用强调色（可配斜体）；
  - 代码标识符、文件名、函数名用下划线或等宽字体。
  - 读者只扫加粗与红字就能复述页面骨架；逐行细读则获得完整论证。两层都必须成立。
- 引用原话、指令、报错时直接嵌入正文并加引号或等宽字体，不要省略成「用户提出修改意见」这类空话。
- 数字必须带口径与时间（「2024 年流水约 7 亿美元，国内移动端占 55% 以上」），孤零零的数字不算证据。

## 逻辑骨架（每页的分析结构必须显性、可复述）
- **固定分析框架复用**：从内容中提炼一组分析维度，做成带冒号的加粗标签前缀，在页内每个小节按同一顺序重复出现。例如「**AI 的行为：**… / **能力边界：**… / **对人的依赖：**…」或「**现象：**… / **根因：**… / **对策：**…」。同一页内框架必须一致，让读者按同一把尺子对比各小节。
- **过程叙事页**：按时间线交替展开「动作 → 反应」（如 Prompt1 → 行为1 → Prompt2 → 行为2 …）；节点之间插入强调色批注句标记转折点（如「用户介入，发现实现不完备，遗漏大量场景」）；页首放元信息块（工具/对象/任务/轮次等键值对），页侧放结果指标面板。
- **评测/对比矩阵页**：满版大表格是页面主体而不是配图。单元格内放多行内容：评分加粗 + 编号小结论列表（1. … 2. …）；失败、异常单元格整体用强调色标出；首列加粗，表头深色底白字。
- 标题写法：「主题 + 判断」（如「编译器场景：开发者必须补全 AI 缺失的全局视野」）或「主题 – 切面」（如「支持 MUSL 使能 HWASAN – 需求」）。正文必须逐条支撑标题。
- 全篇组织：总-分结构。先一页结论总览（可用满版矩阵承载），后续每页深拆一个案例/维度/论点，最后一页给出可执行的启示或建议。
- **主动考虑数据与信息可视化**（参考 SKILL.md 的「数据与信息可视化意识」及 `references/data-information-visualization.md`）：比较、趋势、构成、目标差距、多维关系、流程和里程碑都可能受益于图表或图示，但不做固定映射。Agent 应结合本页论证强度、精确查数需求和整套 deck 节奏自行选择矩阵、图表、流程、结构化文字或混合形式。数据图需标数值、时间、口径与来源，并配「So what」判断；默认可编辑路径不用 `conic-gradient`。

## 推荐版式与量化最低规格（960×540pt，每种版式必须达标）
- **满版评测矩阵**：表格 **7–11 列 × 7–12 行**，行高按内容自适应；信息量大的单元格放「加粗短判断 + 编号小结论（1–3 条）」；失败格标红。表格必须撑满正文区宽度且行数足以排到页底；行数不够就增加对比维度行，或在表格下方补一栏「关键发现」编号列表直至填满。
- **卡片网格深拆**：2×3 或 3×3 网格撑满整个正文区高度，**每张卡片 4–7 行**：加粗小标题 + 3 组以上「**标签：**完整句子」字段（如对比双方各 1–2 句 + 一句差异判断）。六张两行卡片只占半页属于典型不合格输出。
- **三级嵌套深拆页**：3–4 个阶段/分论点小节，每节内按同一组加粗标签维度展开完整句子分析，每节 4–8 行，整页几乎全文字，从分隔线排到脚注线。
- **过程叙事 + 侧栏**：左侧细边框容器装时间线叙事（顶部元信息键值块 → Prompt/行为交替段落 → 红色批注句），节点 4–6 个；右侧窄栏「效果/结论」面板 5 行以上（标签 + 数值）。
- **流程/架构图深拆**：CSS 绘制的多级流程图或架构图占约 55–60%，节点 6 个以上且带标注；旁边一栏逐节点完整句子解释，与节点编号一一对应。
- **左右对照深拆**：左右两栏各装一个多字段实体卡（各 6–10 行），下方横排 1–2 条红色「关键风险/核心疑问」批注条，再加「本页结论」横条收底。两栏只占上半页、下半页留白属于不合格输出。

## 视觉系统
- 纯白背景（#ffffff），零装饰：没有大色块、渐变、图标堆砌和情绪化视觉；**信息本身就是视觉**。
- 顶部标题区一行大字（允许两行），与正文区用细分隔线分开；左上角放章节 kicker（如 03 | 案例深拆），右上角页码。标题区总高度不超过画布的 18%，把空间留给正文。
- 正文承载区占画布 80% 以上。常用分区：单栏全宽（满版表格/嵌套列表）、主区 + 侧栏（约 70% + 25%）、2–3 栏对称拆解。
- 内容容器只用细边框（1–1.5px 实线）和浅灰底面板，不用投影、大圆角、玻璃拟态。
- 页面底部保留来源/口径/假设脚注行（font-size: 9–10px）。

## 配色（CSS 可用）
- 背景：#ffffff（纯白）
- 文字：#1f2937（深灰黑）
- 主色：#1e3a8a（深海军蓝：标题、表头、结构线、编号、标签前缀）
- 强调：#dc2626（红色：失败/风险/转折批注、关键数字，克制使用但该用就用）
- 辅助：#64748b（中灰蓝：副标题、说明文字、脚注）
- 面板：#f1f5f9（浅灰蓝：表头底色、元信息块、侧栏面板、斑马纹）

## 排版（HTML/CSS）
- 正文 11–13px，line-height 1.45–1.6；嵌套列表逐级缩进 14–18px；段间距保持密排（不超过 1.5 倍行距），整页呈现「满而有序」。
- 表格字号 10–12px，单元格 padding 4–8px；单元格内的编号列表用 `margin: 0; padding-left: 14px;` 紧凑排布。
- 结论标题 20–24px 粗体；小节标题与侧栏面板标题 13–14px 加粗。
- 数字、评分、代码用等宽字体（如 "SF Mono", Consolas, monospace）增强报告感。
- 密度的边界：正文不小于 10px、行高不小于 1.35、任何元素不得溢出 960×540pt 画布。**内容装不下时拆成两页；内容装得下但排不满时，按「画布填充硬指标」补内容。**
- 垂直预算心算（写前必做）：标题区 70–95pt + 脚注 20–25pt + 底部安全边距 ≥ 36pt（0.5in），正文区实际约 390–420pt。每个块按「行数 × 字号 × 行高 + padding」估算，所有块合计 ≤ 正文区高度；表格先算「行数 × 行高」再定行数。估超就减行/合栏/拆页，禁止缩字号硬塞。

## CSS 实现要点
- 表格：`border-collapse: collapse;`，表头 `background: #1e3a8a; color: #ffffff;`，斑马纹 `tr:nth-child(even){background: #f1f5f9;}`，单元格 `border: 1px solid #e2e8f0; padding: 4px 8px; vertical-align: top;`。
- 多行单元格：单元格内用 `<ol>`/`<ul>`（`margin: 0; padding-left: 14px; font-size: 10px;`）放编号小结论；失败格 `color: #dc2626; font-weight: 700;` 或整格 `background: #fef2f2;`。
- 标签前缀：`<strong style="color: #1e3a8a;">AI 的行为：</strong>` 后接常规正文，同一页所有小节复用同一组标签。
- 批注条（风险/疑问/转折）：`background: #fef2f2; border-left: 3px solid #dc2626; padding: 6px 10px;`，内文 `color: #dc2626;` 可配斜体。
- 「本页结论」横条：`background: #f1f5f9; border-left: 4px solid #1e3a8a; padding: 8px 12px;`，1–2 个完整句子。
- 元信息块/侧栏面板：`background: #f1f5f9; border: 1px solid #e2e8f0; padding: 10px 12px;`，内部用「加粗标签: 值」逐行排布。
- 流程框：`border: 1.5px solid #1e3a8a; border-radius: 4px; padding: 6px 10px;`，连接线用细 div 或 border 绘制，箭头用 CSS 三角形。
- 布局填满且不破：`body { overflow: hidden; }`；根容器用 `display: flex; flex-direction: column; height: 540pt;`，正文区 `flex: 1; min-height: 0; overflow: hidden;` 并让网格/表格 `height: 100%;` 拉伸——既从结构上杜绝下半页留白，又保证估错时只在容器内裁切、不撑破画布。

## 逐页自检清单（生成每页 HTML 后必须逐条核对，任一不过即重写本页）
1. 正文是否从标题分隔线排到了脚注线，底部空白 ≤ 40pt？
2. 正文字数是否 ≥ 300（目标 450–900）？
3. 每张卡片/每个小节是否 ≥ 4 行、每个表格是否 ≥ 7 行？
4. 每条要点是否都是完整句子并带具体证据（数字/日期/名称/原话）？
5. 页内各小节是否复用了同一组加粗标签维度？
6. 是否有「本页结论」横条或等价的判断性收底？
7. 是否无任何元素溢出 960×540pt、字号 ≥ 10px？垂直预算合计是否 ≤ 正文区高度、文本框底边距画布底部是否 ≥ 0.5in？
8. 本页的文字、表格、图表或图示是否确实最有利于当前论证，而不是沿用固定模板或为了可视化而可视化？

## 避免
- **不留下半页空白**——这是本风格最严重的失败模式；宁可多写一组维度、一行结论，也不留白。
- **不写关键词式短 bullet 页**：只有名词短语的 3–5 条大字要点页是演讲风格，视为不合格输出。
- 不写两行小卡片拼贴页：卡片必须是多字段实体，不是标签贴纸。
- 不做大留白页、巨型 hero 数字页、情绪化金句页；不为「呼吸感」牺牲信息量。
- 不使用装饰性图形、图标墙、大面积色块和渐变；所有视觉元素必须承载信息。
- 不写没有结论的罗列：标题必须给出判断，正文必须支撑标题。
- 不把长内容压成不可读：宁可拆页，不许字号低于 10px、不许行距挤压、不许溢出画布。
- 不省略证据：分析必须落到具体的数字、文件、函数、原话或行为，不许停留在「效果不错」「存在问题」这类空泛判断。

## 输出要求
保留输入内容的信息结构和文字含义，按此风格重绘为统一连贯的 16:9 演示页面；输入内容单薄时必须基于输入展开成完整句子的详尽分析并补齐分析维度，直到满足画布填充硬指标，而不是缩减成短语或留白。所有视觉元素（含表格、流程图、架构图、指标面板）必须用 HTML/CSS 实现，不依赖外部图片资源。
